## 一、 核心体验修复

### 1. 文本渲染：支持 Markdown 与 代码

- **需求描述**：目前大模型输出为纯文本，无法显示表格、加粗、公式及代码块。
- **修改要求**：
  - 集成 `react-markdown` (或 Vue 对应的渲染库)。
  - 配置 `remark-gfm` 插件以支持 GitHub 风格的 Markdown（如表格、任务列表）。
  - 集成 `prismjs` 或 `shiki` 实现代码块的高亮显示，并添加“一键复制”按钮。
  - **数学公式**：集成 `remark-math` 和 `rehype-katex`，确保 $LaTeX$ 公式能正确渲染。

### 2. 交互流优化：即时反馈与生命周期管理

- **需求描述**：解决“发送后内容不显示”和“无法干预生成过程”的问题。
- **修改要求**：
  - **乐观更新 (Optimistic UI)**：点击发送瞬间，立即将用户输入的消息压入消息数组并渲染在界面上，随后显示“模型正在思考”的占位符。
  - **流式响应 (Streaming)**：前端需处理 `Server-Sent Events (SSE)`，让文字像打字机一样实时蹦出，而不是等全部生成完才显示。
  - **中止交互 (Abort Controller)**：在发送请求时绑定 `AbortController`，界面提供“停止生成”按钮，点击后立即中断 API 调用。
  - **二次编辑 (Edit & Resubmit)**：每条用户消息悬浮显示“编辑”图标。点击后进入编辑状态，保存时**自动删除该消息之后的所有对话记录**并触发重新发送。

### 3. 布局重构：三栏独立滚动控制

- **需求描述**：目前目录、主线、衍生线滚动关联，体验混乱。
- **修改要求**：
  - **CSS 结构**：摒弃全局滚动，采用 `H-screen` (100vh) 布局。
  - **容器属性**：
    - **目录栏 (Sidebar)**：固定宽度，`overflow-y: auto`。
    - **主对话流 (Main Canvas)**：弹性宽度或固定比例，`overflow-y: auto`。
    - **衍生窗口 (Parallel Pane)**：绝对定位或 Flex 占据右侧，`overflow-y: auto`。
  - **滚动锚定**：当模型正在输出时，对话流容器需实现“自动滚动到底部”逻辑，但若用户手动向上滚动阅读，则应暂停自动滚动。

## 二、 进阶优化建议 (功能新增)

以下是根据你的“平行衍生”理念进一步挖掘的优化点：

### 1. 局部关联触发 (Contextual Trigger)

- **功能描述**：目前开启衍生模式可能比较生硬。
- **修改要求**：
  - 在支持基于所有已有记忆衍生的基础上，也可以支持实现**“划词触发”**：当用户在主窗口选中一段特定的文字时，弹出一个小气泡按钮“在此开启平行衍生”。
  - **效果**：点击后，右侧衍生窗口开启，并自动带入一个隐藏的 System Prompt：“用户接下来的提问是基于以下这段文字：‘[选中的文本内容]’，请针对此进行详细解答。”

### 2. 线程快照与追溯 (Thread Snapshot)

- **功能描述**：衍生模式产生的对话也是有价值的，需要持久化。
- **修改要求**：
  - 在主对话的记录中，在开启过衍生的位置留下一个“分支标记” (如一个分支图标)。
  - 点击标记，可以重新打开当时那个时间点的衍生窗口，回看当时的追问细节。

### 3. 记忆融合预览 (Memory Merger Preview)

- **功能描述**：用户在关闭衍生模式并选择“融入记忆”时，不确定模型到底记住了什么。
- **修改要求**：
  - 提供一个“记忆摘要”功能。当用户点击合并时，调用模型对衍生线的对话做一次 100 字以内的**摘要提炼**。
  - 将这个摘要作为一条隐形的 `system` 或 `user` 提示词插入主线，既节省了 Token，又保证了记忆的准确性。

### 4. 全球模型对比 (Model Arena in Parallel)

- **功能描述**：利用衍生窗口做横向对比。
- **修改要求**：
  - 允许衍生窗口选择**与主窗口不同的模型**。
  - **场景**：主窗口用 GPT-4o 写代码，衍生窗口用 DeepSeek 专门解释这段代码的逻辑，实现多模型协同办公。

## 三、 推荐技术架构实现参考

**前端状态管理建议：**

针对你的“衍生线”逻辑，建议在 `store` 中这样定义数据结构：

JavaScript

```
{
  sessions: [
    {
      id: "session_1",
      mainThread: [msg1, msg2, msg3], // 主线消息
      derivationThreads: {
        "msg2_offset": [dev_msg1, dev_msg2] // 挂载在第二条消息位置的衍生线
      }
    }
  ]
}
```

**CSS 快速修复（针对独立滚动）：**

CSS

```
/* 确保父容器不滚动 */
.app-container {
  display: flex;
  height: 100vh;
  overflow: hidden;
}

/* 每一个子列独立滚动 */
.sidebar, .main-thread, .parallel-pane {
  height: 100%;
  overflow-y: auto;
  /* 自定义滚动条美化 */
}
```
